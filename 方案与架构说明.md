# 图像识别方案说明（汇报用）

本文档汇总**当前系统架构**，以及 **Otsu、自适应二值、高斯模糊**、**CLAHE**、**整图分水岭** 等概念与在本项目中的用法，便于向上司或同事介绍解决方案。

---

## 一、系统架构

### 1.1 总体形态

- **单文件桌面应用**：业务与界面集中在 `main.py`（约 610 行），未拆多包。
- **无 Web 服务、无数据库**。
- **依赖**（见 `requirements.txt`）：
  - **PyQt6**：界面
  - **opencv-python-headless**：图像处理
  - **NumPy**：数组与数值计算

### 1.2 逻辑分层

| 层次 | 职责 |
|------|------|
| **入口** | `main()` 创建 `QApplication`，实例化 `MainWindow` 并进入事件循环 |
| **界面** | `MainWindow`：按钮、单选（三种模式）、复选框、三个可滚动预览区、状态说明 |
| **显示桥接** | `numpy_bgr_to_qpixmap`：将 OpenCV 的 BGR/灰度 `ndarray` 转为 `QPixmap`（含缩放） |
| **算法** | 二值化、分水岭、霍夫圆、轮廓与拟合等，以模块级函数 + 类方法组织，不依赖 Qt |

### 1.3 数据流（简述）

1. **打开图片**：`cv2.imread` → 保存彩色 `_bgr` 与灰度 `_gray`；左侧显示原图。
2. **中间「处理结果」**：随单选切换刷新，展示 **Otsu 二值 / 自适应二值 / 9×9 模糊灰度**（与后续识别输入一致）。
3. **「轮廓与拟合」** 分两条主路径：
   - **灰度·Hough**：在可选 CLAHE 后的灰度上模糊 → 主 `HoughCircles` → 对明显过大的圆做 ROI 内二次霍夫 → 去重 → 在原图上画圆。
   - **二值模式**（自适应或 Otsu）：生成掩膜 → 必要时自动反相 →  
     - 若勾选**整图分水岭**：全图分水岭后按连通块找轮廓与最小外接圆；  
     - 否则 **方案 A**：`findContours`，对面积过大的连通域在 ROI 内做**局部分水岭**拆成多个圆，否则单轮廓上画轮廓 + 最小外接圆 +（可选）椭圆拟合。

状态栏文案汇总当前模式、掩膜来源、CLAHE/反相及轮廓与圆数量等。

### 1.4 小结

架构为 **PyQt6 单窗口 + 过程式 OpenCV 管线**：无严格 MVC 分层，状态主要在 `MainWindow` 的成员与控件上。适合原型与小工具；若后续扩展，可再拆为 `ui/`、`pipeline/` 等模块。

## 二、高斯模糊、Otsu、自适应二值：是什么与区别

### 2.1 高斯模糊（Gaussian Blur）

**是什么：**  
对图像做**平滑**：每个像素用周围像素的**加权平均**替换，权重按距离呈高斯分布（离中心越远权重越小）。

**作用：**

- 抑制**噪声**、缓和边缘锯齿。
- 为后续**阈值**或**霍夫圆**提供更稳定的输入。

**注意：**  
高斯模糊不是「分割」或「二值化」本身，而是**预处理**。模糊过强会丢失细节。

**在本项目中的使用：**

- 自适应二值前对灰度使用 `5×5` 高斯模糊。
- 「灰度·Hough」路径对灰度使用 `9×9` 模糊，作为霍夫圆检测的输入。

---

### 2.2 Otsu（大津法 / Otsu 阈值）

**是什么：**  
**全局**二值化：整幅图只使用**一个**阈值 \(T\)，将像素分为「暗」「亮」两类。Otsu 根据**灰度直方图**自动选取 \(T\)，使两类之间的**类间方差**最大（即常见的「自动最优分割阈值」思路）。

**特点：**

- **优点：** 全自动、实现简单；在**光照均匀、前景与背景对比清晰**时效果很好。
- **缺点：** 假设整图可用**同一**明暗分界；若**左亮右暗**、存在大面积阴影，单一全局阈值容易**顾此失彼**。

**在本项目中的使用：**  
「二值·Otsu」在（可选 CLAHE 后的）灰度上做一次全局 Otsu，得到**整图共用**的二值掩膜，再交给轮廓 / 分水岭等后续步骤。

---

### 2.3 自适应阈值（Adaptive Threshold）

**是什么：**  
**局部**二值化：**每个像素（或小区域）使用各自的阈值**，通常由该点邻域内的统计量（如局部均值或高斯加权均值）减去常数 \(C\) 得到。本项目使用 `ADAPTIVE_THRESH_GAUSSIAN_C`，即阈值来自邻域的**高斯加权**统计，比简单矩形块更平滑。

**特点：**

- **优点：** 适应**光照不均**、局部对比变化（例如显微镜、血涂片等场景）。
- **缺点：** 需设定**块大小（block size）**；块过大接近全局，块过小可能产生**块状伪影**或把噪声当成结构。

**在本项目中的使用：**  
「二值·自适应」按图像尺寸计算 block，在模糊后的灰度上做自适应二值，得到**随位置变化**的掩膜，后续同样走轮廓与方案 A 局部分割等流程。

---

### 2.4 三者关系（一句话对照表）

| 名称 | 类型 | 一句话 |
|------|------|--------|
| **高斯模糊** | 预处理 | 平滑降噪，为后续阈值或霍夫做准备。 |
| **Otsu** | 全局二值 | 整图**一个**自动阈值，适合**光照均匀、明暗分界清楚**。 |
| **自适应二值** | 局部二值 | **各区域分别**算阈值，适合**光照不均、局部对比差**。 |

**关系说明：**  
三者并非简单三选一：**高斯模糊**常与 **Otsu** 或 **自适应** 搭配使用；**Otsu** 与 **自适应** 才是两种不同的**二值化策略**（全局 vs 局部）。

## 三、CLAHE 与整图分水岭

### 3.1 CLAHE 是什么

**CLAHE** = **Contrast Limited Adaptive Histogram Equalization**（**限制对比度的自适应直方图均衡**）。

**直觉：**

- 普通**直方图均衡**整幅图一起拉对比度，容易把**噪声也拉爆**，出现过亮/过暗斑块。
- **自适应**：把图像划成许多**小块（tile）**，**每块各自**做均衡，从而改善**局部**对比度（例如一侧偏暗、一侧偏亮时，暗区也能拉开层次）。
- **对比度限制（Clip Limit）**：对直方图做裁剪与重分配，避免某一块均衡得「过狠」导致**噪声被异常放大**。

**在本项目中的使用：**  
界面勾选「CLAHE 预处理」后，在灰度上使用 OpenCV `createCLAHE`（`clipLimit=2.0`，`tileGridSize=(8, 8)`），得到的灰度作为后续**二值预览、掩膜、霍夫**等步骤的统一输入。适合**整体发灰、局部对比不足**的图像（如血涂片等可尝试）。

**与分水岭的关系：**  
CLAHE 作用于**预处理**（改善输入灰度）；分水岭作用于**二值之后的分割**，二者层次不同、可独立开关。

---

### 3.2 整图分水岭是什么

**分水岭（Watershed）**是一种基于「灰度地形」的分割思路：把图像看成起伏地形，从种子区域向外扩展，不同区域相遇处形成边界。OpenCV 中常用的是 **基于标记的分水岭（marker-based watershed）**：由二值结果得到**前景 / 背景 / 未知**等标记，再在**彩色引导图**上调用 `cv2.watershed`，输出每个像素的**区域标签**。

**「整图」在本项目中的含义：**  
勾选「整图分水岭」时，在**整张图**尺度上对当前二值掩膜做距离变换与标记构建，再对**整幅**彩色图（副本）执行一次 `cv2.watershed`，随后按每个实例取轮廓并拟合最小外接圆。即分割在**全图一次完成**，而不是只在某个大轮廓的 ROI 内操作。

**风险：**  
整图分水岭**容易过分割**（同一目标被切成多块、或细节被切碎），因此界面说明中建议**默认关闭**，需按数据谨慎评估。

---

### 3.3 整图分水岭 vs 方案 A（局部分水岭）

| 项目 | 整图分水岭 | 方案 A（默认） |
|------|------------|----------------|
| **范围** | 全图一次分水岭 | 先 `findContours`，仅对**面积过大**的连通域在其 **ROI** 内做局部分水岭 |
| **与另一选项关系** | 开启时**不再使用**方案 A 的局部分割逻辑 | 与「整图分水岭」互斥（二选一流程） |
| **适用直觉** | 希望全局统一拆分粘连区域时尝试 | 只对「异常大团」局部拆分，更保守 |

## 四、向上司汇报时可用的方案表述

1. **输入**：读入图像，可选 **CLAHE**（见第三节）提升局部对比，减轻发灰与光照不均对后续步骤的影响。
2. **两条二值路线**：
   - **Otsu**：场景稳定、光照均匀时，简单可靠。
   - **自适应**：光照或背景不均匀时，用局部阈值提高鲁棒性。
3. **另一条路线（灰度·Hough）**：不做二值掩膜，在**模糊后的灰度**上直接做霍夫圆，适合边缘呈圆形、未必适合硬二值的图像；并对特别大的圆做 **ROI 二次检测**。
4. **后处理**：在二值掩膜上找轮廓。默认用**方案 A**：仅对**过大的连通域**在 ROI 内做**局部分水岭**拆成多个圆；若开启**整图分水岭**，则改为全图分水岭后再按实例拟合（力度更大、更易过切，默认关）。

若被问「为什么有两种二值化」，可答：**全局 Otsu 省事但不抗不均匀光照；自适应抗光照变化但依赖块大小等参数、计算略重。** **高斯模糊**服务于稳定输入，不是与二者并列的第三种「识别算法」。
